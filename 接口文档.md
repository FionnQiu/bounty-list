# 后端API接口文档

## 1. 健康检查接口

### GET /api/health

- **功能**：检查服务是否正常运行
- **认证**：不需要
- **请求参数**：无
- **响应格式**：
  ```json
  {
    "success": true,
    "message": "服务运行正常"
  }
  ```

## 2. 认证相关接口

### POST /api/auth/register

- **功能**：用户注册
- **认证**：不需要
- **请求参数**：
  | 字段名      | 类型     | 必填 | 描述  |
  | :------- | :----- | :- | :-- |
  | username | string | 是  | 用户名 |
  | password | string | 是  | 密码  |
  | nickname | string | 是  | 昵称  |
- **响应格式**：
  ```json
  {
    "success": true,
    "message": "注册成功",
    "data": {
      "token": "JWT令牌",
      "user": {
        "id": 1,
        "username": "chenxi",
        "nickname": "陈曦",
        "avatarUrl": "头像URL",
        "bio": "个人简介"
      }
    }
  }
  ```

### POST /api/auth/login

  | 字段名      | 类型     | 必填 | 描述  |
  | :------- | :----- | :- | :-- |
  | username | string | 是  | 用户名 |
  | password | string | 是  | 密码  |
- **响应格式**：
  ```json
  {
    "success": true,
    "message": "登录成功",
    "data": {
      "token": "JWT令牌",
      "user": {
        "id": 1,

  }
  ```

  ```json
    "data": {
  ```

## 4. 悬赏相关接口

### GET /api/bounties

- **功能**：获取悬赏列表
- **认证**：需要（Bearer Token）
- **请求参数**：
  | 字段名        | 类型     | 必填 | 描述                                     |
  | :--------- | :----- | :- | :------------------------------------- |
  | page       | number | 否  | 页码，默认1                                 |
  | pageSize   | number | 否  | 每页数量，默认12                              |
  | keyword    | string | 否  | 搜索关键词                                  |
  | categoryId | number | 否  | 分类ID                                   |
  | status     | string | 否  | 状态（open/in\_progress/completed/closed） |
  | sort       | string | 否  | 排序方式（latest/reward\_desc/reward\_asc）  |
- **响应格式**：
  ```json
  {
    "success": true,
    "message": "获取悬赏列表成功",
    "data": {
      "items": [
        // 悬赏列表
      ],
  ```

### POST /api/bounties

- **功能**：发布悬赏
- **认证**：需要（Bearer Token）
- **请求参数**：
  | 字段名          | 类型     | 必填 | 描述          |

## 6. 认证方式

所有需要认证的接口都需要在请求头中添加：

```
Authorization: Bearer {token}
```

其中 `{token}` 是登录或注册成功后返回的 JWT 令牌。

## 7. 错误处理

所有接口的错误响应格式统一为：

```json
{
  "success": false,
  "message": "错误信息",
  "data": {
    "error": "错误详情"
  }
}
```

常见的错误状态码：

- 400: 请求参数错误
- 401: 未登录或 token 失效
- 404: 资源不存在
- 409: 资源冲突（如用户名已存在）
- 500: 服务器内部错误

## 8. 前端调用示例

```javascript
// 使用 axios 调用接口
import axios from 'axios';

// 创建 axios 实例
const apiClient = axios.create({
  baseURL: '/api',
  headers: {
    'Content-Type': 'application/json'
  }
});

// 添加认证拦截器
apiClient.interceptors.request.use(config => {
  const token = localStorage.getItem('bounty_token');
  if (token) {
    config.headers.Authorization = `Bearer ${token}`;
  }
  return config;
});

// 登录示例
async function login(username, password) {
  try {
    const response = await apiClient.post('/auth/login', {
      username,
      password
    });
    return response.data;
  } catch (error) {
    throw error.response.data;
  }
}

// 获取悬赏列表示例
async function getBounties(params) {
  try {
    const response = await apiClient.get('/bounties', { params });
    return response.data;
  } catch (error) {
    throw error.response.data;
  }
}
```
